Lesson-by-Lesson Focus

Lesson 13 — Normal Distribution

TipTakeaway
  • Normal distribution: \(X \sim N(\mu, \sigma^2)\)
  • Standardization: \(z = \frac{x - \mu}{\sigma}\)
  • Use normal probabilities via z-table lookup
  • Find normal quantiles for given tail areas
  • Assess plausibility using normal models
ImportantAssessment Focus
  • Standardization work shown: Cadets must write out \(z = \frac{x - \mu}{\sigma}\) and show the substitution. No credit for jumping straight to the answer.
  • Forgetting to standardize: The most common error — using \(x\) values directly in the z-table.
  • Sketch the curve: Encourage drawing the Normal curve, marking \(\mu\) and the value of interest. This prevents sign errors and “wrong tail” mistakes.

Lesson 14 — Exponential Distribution

TipTakeaway
  • Specify distribution: \(T \sim \text{Exp}(\lambda)\)
  • Rate parameter \(\lambda\) vs. mean \(1/\lambda\)
  • CDF: \(F(x) = 1 - e^{-\lambda x}\)
  • \(P(T > t) = e^{-\lambda t}\)
  • Memoryless property
ImportantAssessment Focus
  • Rate vs mean confusion: Confusing the exponential rate \(\lambda\) with the Poisson \(\lambda\) (they’re related but used differently). If \(\lambda = 0.02\) failures/hour, the mean time is \(1/0.02 = 50\) hours.
  • Memoryless property: Cadets should recognize and apply \(P(T > s + t \mid T > s) = P(T > t)\). This is a favorite WPR question.
  • Poisson-Exponential connection: If events occur at rate \(\lambda\) per unit time (Poisson), the time between events is \(\text{Exp}(\lambda)\).

Lesson 16 — WPR 1 (175 pts)

ImportantWPR 1 Grading Focus
  • Distribution identification: Each problem should begin with identifying the correct distribution and stating its parameters. Partial credit if the distribution is correct but the calculation is wrong.
  • Conditions/assumptions stated: Even on the WPR, cadets should note conditions (e.g., “assuming independence” or “since \(n\) is fixed and trials are binary, this is Binomial”).
  • Interpretation in context: Final answers should be complete sentences referencing the problem scenario, not just a number.
  • Common deductions: Off-by-one errors in complement calculations, using \(\lambda\) from Poisson in an exponential context without adjustment, forgetting continuity.
  • Rubric calibration: Before grading, align across sections on partial credit for each problem. Grade one problem at a time across all exams, not one exam at a time.

TEE-Style Question

CautionWhen All Is Said and Done…

Cadets should be able to answer a question like this:

The time (in hours) until a generator fails follows an Exponential distribution with \(\lambda = 0.02\) failures per hour.

  1. What is the expected time until failure?

  2. What is the probability the generator lasts more than 60 hours?

  3. Given that the generator has already been running for 40 hours, what is the probability it lasts at least 20 more hours? What property does this illustrate?

  4. Suppose ACFT scores are Normally distributed with \(\mu = 450\) and \(\sigma = 55\). What proportion of soldiers score between 400 and 520?

Teaching Demo

NoteSuggested Classroom Demonstration

Normal Distribution — Z-Score Walkthrough (Lesson 13)

Use a military example: “ACFT scores for a population of soldiers are approximately Normal with \(\mu = 450\) and \(\sigma = 50\). What proportion score above 520?”

Walk through on the board:

  1. Sketch the Normal curve, mark \(\mu = 450\) and \(x = 520\)
  2. Standardize: \(z = \frac{520 - 450}{50} = 1.4\)
  3. Use the Z-table: \(P(Z \leq 1.4) = 0.9192\)
  4. Therefore: \(P(X > 520) = 1 - 0.9192 = 0.0808\)
  5. Interpret: “About 8.1% of soldiers score above 520.”

Then show the same calculation in R: 1 - pnorm(520, mean = 450, sd = 50). Connect the table approach to the computational approach.

Review Session Structure (Lesson 15): Build a distribution summary table with cadets — one row per distribution (Binomial, Poisson, Exponential, Normal) with columns for: type, parameters, E(X), Var(X), key conditions, R functions.
