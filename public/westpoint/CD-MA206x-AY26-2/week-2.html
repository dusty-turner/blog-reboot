Lesson-by-Lesson Focus

Lesson 5 — Exploratory Data Analysis Lab

TipTakeaway
  • Lab day to apply everything from Lessons 1–4
  • Report IQR with median (for skewed data or outliers)
  • Report standard deviation with mean (for symmetric data)
ImportantAssessment Focus

EDA Lab (25 pts): Look for clear graph titles/labels, appropriate graph types for the data, and written interpretations that go beyond “the histogram shows the data.” Cadets should connect their visualizations to a story about the data. Watch for rendering issues — code chunks set to eval = FALSE, missing packages, broken file paths. Deduct if output is missing.

Lesson 6 — Probability Basics

TipTakeaway
  • Sample spaces, events, and set operations
  • The three Kolmogorov Axioms of probability
  • Complement rule: \(P(A^c) = 1 - P(A)\)
  • Addition rule (Inclusion-Exclusion): \(P(A \cup B) = P(A) + P(B) - P(A \cap B)\)
ImportantAssessment Focus
  • Probability notation: Watch for correct use of \(P(A \cap B)\), \(P(A \mid B)\), and \(P(A \cup B)\). Sloppy notation now becomes a major problem later.
  • Axiom application: Can cadets identify which axiom/rule applies in a given scenario?
  • Complement errors: Watch for \(P(A^c) = P(A) - 1\) instead of \(1 - P(A)\).

Lesson 7 — Conditional Probability

TipTakeaway
  • Conditional probability: \(P(A \mid B) = \frac{P(A \cap B)}{P(B)}\)
  • Multiplication rule for \(P(A \cap B)\)
  • Law of Total Probability
  • Bayes’ Theorem: flipping conditional probabilities
ImportantAssessment Focus
  • Bayes’ Rule setup: Cadets should use tree diagrams to organize information before computing. The formula alone without structure leads to errors.
  • Direction of conditioning: Watch for confusing \(P(A|B)\) with \(P(B|A)\) — the prosecutor’s fallacy. This is a TEE-level error.
  • Law of Total Probability: Can cadets partition a sample space and sum over branches?

Lesson 8 — Independence

TipTakeaway
  • The multiplication principle for counting outcomes
  • Permutations (ordered) vs. Combinations (unordered)
  • With vs. without replacement
  • Independence: \(P(A \cap B) = P(A) \cdot P(B)\)
  • Multiplication rule for independent events
ImportantAssessment Focus
  • Independence verification: Cadets should verify independence using \(P(A \cap B) = P(A) \cdot P(B)\), not just assert it.
  • Mutually exclusive vs independent: These are not the same (they’re actually opposites for non-trivial events). This is a very common misconception — test it directly.
  • Permutations vs combinations: “Does order matter?” is the key question. Watch for cadets using the wrong formula.

TEE-Style Question

CautionWhen All Is Said and Done…

Cadets should be able to answer a question like this:

A factory produces circuit boards using two machines. Machine A produces 60% of boards and Machine B produces 40%. The defect rate for Machine A is 3% and for Machine B is 5%.

  1. A board is selected at random and found to be defective. What is the probability it came from Machine A?

  2. Are the events “defective” and “from Machine A” independent? Show your work.

  3. Are they mutually exclusive? Explain.

Teaching Demo

NoteSuggested Classroom Demonstration

Tree Diagrams for Bayes’ Rule (Lesson 7)

Use a classic medical testing example: “A disease affects 1% of the population. A test has 95% sensitivity and 90% specificity. If you test positive, what’s the probability you actually have the disease?”

Draw the full tree diagram on the board:

  1. Branch on disease status: \(P(D) = 0.01\), \(P(D^c) = 0.99\)
  2. Branch on test result from each
  3. Compute \(P(D \mid +)\) using the tree

The answer (~8.8%) always surprises cadets and makes Bayes’ Rule stick.

Independence vs. Mutually Exclusive (Lesson 8): Roll two dice in class. Ask: “Are the outcomes independent?” (yes). “Are they mutually exclusive?” (no — both can happen). Then: “If I draw a card, are ‘red’ and ‘heart’ mutually exclusive?” (no). “Independent?” (no — knowing it’s red changes \(P(\text{heart})\)).
