Lesson 24: Guided Example — Time Series & Forecasting

Overview

When data has a time dimension, new structure emerges: trends, seasonal patterns, and autocorrelation. Time series analysis gives you tools to decompose these patterns, visualize them, and project them into the future.

This lesson uses the tsibble, feasts, and fable packages — the tidy time series ecosystem. You will visualize temporal patterns, decompose a series into trend, season, and remainder, fit forecasting models, and evaluate their accuracy.

Objectives

NoteBy the end of this lesson, you will be able to:
  • Describe the components of a time series: trend, seasonality, residual
  • Create and work with tsibble objects
  • Visualize time series patterns with autoplot, gg_season, and gg_subseries
  • Decompose a series using STL
  • Fit forecasting models: ETS, ARIMA, NAIVE, SNAIVE
  • Evaluate forecast accuracy with MAE, RMSE, MAPE, and MASE
  • Check residual diagnostics with gg_tsresiduals()

Setup

library(tidyverse)
library(tsibble)
library(feasts)
library(fable)

# Australian tourism data (built into tsibble)
tourism
# We'll focus on total domestic tourism in Australia
aus_total <- tourism |>
  summarize(Trips = sum(Trips))

aus_total

Time Series Structure

Every time series can be thought of as the combination of three components:

Component What it is Example
Trend Long-term increase or decrease Steadily rising tourism over decades
Seasonality Regular, repeating pattern More travel in summer, less in winter
Remainder Everything else (noise, shocks) A pandemic, a one-time event

Visualization

The Time Plot

# Basic time plot
aus_total |>
  autoplot(Trips) +
  labs(title = "Total Domestic Tourism — Australia",
       y = "Trips (thousands)") +
  theme_minimal()

Seasonal Plot

# Each year overlaid — reveals seasonal pattern
aus_total |>
  gg_season(Trips) +
  labs(title = "Seasonal Plot: Domestic Tourism",
       y = "Trips (thousands)") +
  theme_minimal()

Subseries Plot

# Average for each season across years
aus_total |>
  gg_subseries(Trips) +
  labs(title = "Subseries Plot: Domestic Tourism",
       y = "Trips (thousands)") +
  theme_minimal()
TipThree Plots, Three Questions
  • Time plot: What is the overall trend?
  • Seasonal plot: Is there a repeating pattern within each year?
  • Subseries plot: Is the seasonal pattern changing over time?

STL Decomposition

STL (Seasonal and Trend decomposition using Loess) splits the series into its three components.

# Decompose
aus_total |>
  model(STL(Trips)) |>
  components() |>
  autoplot() +
  labs(title = "STL Decomposition of Australian Tourism") +
  theme_minimal()
NoteReading the Decomposition

The gray bars on the left show the relative scale of each panel. If the remainder is large relative to the trend, the series is noisy. If the seasonal component is large, seasonality dominates.

Forecasting Models

We will fit four models and compare them.

# Split into training and test
train <- aus_total |> filter(Quarter <= yearquarter("2014 Q4"))
test  <- aus_total |> filter(Quarter >  yearquarter("2014 Q4"))

# Fit multiple models
fits <- train |>
  model(
    naive   = NAIVE(Trips),
    snaive  = SNAIVE(Trips),
    ets     = ETS(Trips),
    arima   = ARIMA(Trips)
  )

fits

What Each Model Does

Model Idea Best for…
NAIVE Tomorrow = today Baseline; no pattern
SNAIVE Tomorrow = same day last year Strong seasonality, no trend
ETS Exponential smoothing (error, trend, season) Automatic trend + season
ARIMA Autoregressive integrated moving average Flexible; handles many patterns

Generating Forecasts

# Forecast the test period
fc <- fits |>
  forecast(h = nrow(test))

# Plot forecasts against actuals
fc |>
  autoplot(aus_total, level = NULL) +
  labs(title = "Forecasts vs Actuals",
       y = "Trips (thousands)") +
  theme_minimal()
# With prediction intervals (just ETS)
fits |>
  select(ets) |>
  forecast(h = nrow(test)) |>
  autoplot(aus_total) +
  labs(title = "ETS Forecast with Prediction Intervals",
       y = "Trips (thousands)") +
  theme_minimal()

Accuracy Metrics

# Compare models on the test set
accuracy(fc, aus_total)

What Each Metric Tells You

Metric Formula Intuition Notes
MAE Average absolute error Same units as the data
RMSE Root mean squared error Penalizes large errors more
MAPE Mean absolute percentage error Scale-free; fails when actuals near zero
MASE MAE relative to naive forecast Best for comparison; < 1 means better than naive
WarningNo Single Best Metric

MASE is generally the most reliable comparison metric. MAPE is intuitive but breaks down near zero. RMSE penalizes big misses. Report multiple metrics and let the context guide interpretation.

ARIMA Basics

ARIMA(p, d, q) has three components:

Component Meaning What it captures
AR(p) Autoregressive Today depends on the last p values
I(d) Integrated Differencing to remove trend
MA(q) Moving average Today depends on the last q forecast errors

For seasonal data, ARIMA adds seasonal terms: ARIMA(p,d,q)(P,D,Q)[m].

Auto ARIMA

You do not need to choose p, d, and q manually. ARIMA() in fable selects them automatically.

# Let R choose the best ARIMA specification
arima_fit <- train |>
  model(ARIMA(Trips))

# What did it choose?
report(arima_fit)
TipPractical Advice

Use auto ARIMA as your default. It searches over combinations of p, d, q and picks the model with the best AIC. You can override if you have domain knowledge, but the automatic selection is a strong starting point.

Residual Diagnostics

A good forecasting model should have residuals that look like white noise: no pattern, no autocorrelation, roughly normal.

# Residual diagnostics for the ETS model
fits |>
  select(ets) |>
  gg_tsresiduals() +
  labs(title = "Residual Diagnostics: ETS Model")
NoteWhat to Check

gg_tsresiduals() produces three panels:

  1. Time plot of residuals: Should look like random noise around zero
  2. ACF plot: Bars should be within the blue dashed bands (no significant autocorrelation)
  3. Histogram: Should be roughly bell-shaped

If residuals show patterns, the model is missing structure in the data.

# Residual diagnostics for the ARIMA model
fits |>
  select(arima) |>
  gg_tsresiduals() +
  labs(title = "Residual Diagnostics: ARIMA Model")

The Full Time Series Workflow

  1. Visualize: Time plot, seasonal plot, subseries plot
  2. Decompose: STL to understand trend, season, and remainder
  3. Split: Training and test sets (time-ordered — no random splitting)
  4. Model: Fit multiple models (at minimum: a naive baseline + one smart model)
  5. Forecast: Generate predictions for the test period
  6. Evaluate: Compare accuracy metrics (MASE, RMSE, MAE)
  7. Diagnose: Check residuals for leftover patterns
  8. Communicate: Plot forecasts with intervals; be honest about uncertainty
WarningTime Series Splitting Is Different

You cannot randomly split time series data. The test set must come after the training set in time. Randomly shuffling would let the model peek at the future.
