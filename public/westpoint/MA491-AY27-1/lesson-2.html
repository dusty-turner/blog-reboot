Lesson 2: Linux Deep Dive, SSH, & Installing RStudio

Objectives

NoteBy the end of this lesson, you will be able to:
  • Use core Linux commands: pwd, ls, cd, mkdir, cp, mv, rm, cat, nano
  • Read and modify file permissions with chmod and chown
  • SSH into the Raspberry Pi from your laptop
  • Install R and RStudio Server on the Pi via the command line
  • Access RStudio Server from your laptop’s browser

Before Class

  • Make sure your Pi is powered on and connected to the network
  • Have your Pi’s IP address handy (run hostname -I on the Pi if you forgot)
  • Bring your laptop — you will need it today

Lesson Outline

1. Linux Command Line Fundamentals (20 min)

Last lesson you used pwd, ls, and cd. Today we go deeper. These commands are the bread and butter of working in a terminal. You will use them every single day.

Navigation and File Operations

pwd                  # Print working directory — where am I?
ls                   # List files in current directory
ls -la               # List ALL files with details (permissions, size, date)
cd Documents         # Change into a directory
cd ..                # Go up one level
cd ~                 # Go to your home directory
cd /                 # Go to the root of the file system

Creating, Copying, Moving, Deleting

mkdir myproject      # Create a directory
mkdir -p a/b/c       # Create nested directories in one shot
cp file.txt backup.txt          # Copy a file
cp -r mydir/ mydir_backup/      # Copy a directory (recursive)
mv old_name.txt new_name.txt    # Rename a file
mv file.txt ~/Documents/        # Move a file to another directory
rm file.txt          # Delete a file (permanent — no trash can)
rm -r mydir/         # Delete a directory and everything in it
Warningrm Is Permanent

There is no undo, no trash can, no recycle bin. rm deletes files immediately and permanently. Double-check before you press Enter, especially with rm -r.

Viewing and Editing Files

cat file.txt         # Print entire file contents to the screen
head -5 file.txt     # Show the first 5 lines
tail -5 file.txt     # Show the last 5 lines
nano file.txt        # Open file in a simple text editor

nano is a terminal text editor. Use the keyboard shortcuts at the bottom of the screen: Ctrl+O to save, Ctrl+X to exit.

Practice

cd ~
mkdir linux-practice
cd linux-practice
echo "Hello from the command line" > greeting.txt
cat greeting.txt
cp greeting.txt greeting_backup.txt
mv greeting_backup.txt archive.txt
ls
rm archive.txt
ls
TipAI Assist

If you forget a command or its flags, ask your AI assistant. “How do I copy a directory recursively in Linux?” is a perfectly good question. The point is not to memorize every flag — it’s to know what’s possible and where to look.

2. File Permissions (15 min)

Every file and directory in Linux has permissions that control who can read, write, or execute it.

Reading Permissions

ls -la
-rw-r--r-- 1 pi pi 1024 Jan 10 09:00 notes.txt
drwxr-xr-x 2 pi pi 4096 Jan 10 09:00 scripts/

Break down -rw-r--r--:

-     rw-    r--    r--
|     |      |      |
type  owner  group  everyone else

r = read (4)   w = write (2)   x = execute (1)   - = no permission

Changing Permissions: chmod

chmod +x script.sh        # Add execute permission for everyone
chmod 755 script.sh        # owner: rwx, group: r-x, others: r-x
chmod 644 notes.txt        # owner: rw-, group: r--, others: r--
chmod 600 secret.txt       # owner: rw-, nobody else can touch it

The numbers work by adding: r=4, w=2, x=1. So 7 = read+write+execute, 5 = read+execute, 4 = read only.

Changing Ownership: chown

sudo chown pi:pi file.txt          # Set owner and group to pi
sudo chown -R pi:pi myproject/     # Recursively change ownership

You’ll need chown later when installing software that creates files owned by root.

Practice

echo "top secret info" > secret.txt
ls -la secret.txt
chmod 600 secret.txt
ls -la secret.txt          # Should show -rw------- (only you can read/write)

echo '#!/bin/bash' > hello.sh
echo 'echo "Hello MA491"' >> hello.sh
./hello.sh                 # Permission denied
chmod +x hello.sh
./hello.sh                 # Hello MA491

3. SSH: Remote Access (15 min)

SSH (Secure Shell) lets you control your Pi from your laptop over the network. Once this works, you unplug the monitor and keyboard. Your Pi becomes a headless server.

Find Your Pi’s IP Address

On the Pi (with the monitor still connected):

hostname -I

Write down the IP address (e.g., 192.168.1.42).

Connect From Your Laptop

Open a terminal on your laptop:

  • Mac/Linux: open Terminal
  • Windows: open PowerShell or Git Bash
ssh your-username@192.168.1.42

The first time, you’ll see a fingerprint warning. Type yes. Enter your password.

You are now typing commands on your Pi from your laptop. Everything you run happens on the Pi.

Go Headless

Once SSH is working:

  1. Unplug the monitor
  2. Unplug the keyboard and mouse
  3. Your Pi only needs power and a network connection

Welcome to server management.

WarningCan’t Connect?

Common fixes:

  • Re-check the IP address (hostname -I on the Pi)
  • Make sure your laptop is on the same network
  • Try ping 192.168.1.42 from your laptop
  • Make sure SSH is enabled: sudo systemctl status ssh on the Pi

4. Install R and RStudio Server (20 min)

Now for the payoff. You are going to install R and RStudio Server on your Pi, then access RStudio from your laptop’s web browser. This is exactly how data science teams work in the real world — a server runs the computation, and you interact through a browser.

Install R

sudo apt update
sudo apt install -y r-base r-base-dev

This takes several minutes. Verify when it’s done:

R --version

Install RStudio Server

RStudio Server is the web-based version of RStudio. It runs on your Pi and you access it from any browser on the network.

# Install dependencies
sudo apt install -y gdebi-core

# Download RStudio Server for ARM (Pi's architecture)
# Check https://posit.co/download/rstudio-server/ for the latest version
wget https://download2.rstudio.org/server/jammy/arm64/rstudio-server-2024.12.1-563-arm64.deb

# Install it
sudo gdebi -n rstudio-server-2024.12.1-563-arm64.deb
TipAI Assist

The exact download URL changes with each RStudio release. If the URL above doesn’t work, ask your AI assistant: “What is the latest RStudio Server download URL for ARM64 Debian?” It will find the current link.

Verify RStudio Server Is Running

sudo systemctl status rstudio-server

You should see active (running).

Access RStudio From Your Laptop

  1. On your laptop, open a web browser
  2. Navigate to http://YOUR_PI_IP:8787 (e.g., http://192.168.1.42:8787)
  3. Log in with your Pi username and password

You should see the full RStudio IDE in your browser. You are now writing R code that runs on the Pi.

# Test it — run this in the RStudio console
print("RStudio Server is running on my Raspberry Pi!")
Sys.info()["machine"]   # Should show "aarch64" (ARM architecture)
NoteWhy RStudio Server?

Running RStudio as a server means:

  • Your Pi does the computation (you can close your laptop and it keeps running)
  • Anyone on the network can access it from a browser
  • You can run long jobs without tying up your laptop
  • This is how cloud-based data science platforms (Posit Cloud, JupyterHub) work

5. Wrap-Up (5 min)

Quick status check:

  • Can you SSH into your Pi from your laptop?
  • Is the monitor unplugged?
  • Can you open RStudio Server in your browser at Pi_IP:8787?

If any of these aren’t working, stay after class or troubleshoot with AI assistance before next lesson. Everything from here on assumes you have SSH and RStudio Server working.
