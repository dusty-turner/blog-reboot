Lesson 15: Building a Portfolio — Personal Webpage on Netlify

Why Build a Portfolio?

You have two deployed projects now — NYC 311 and your semi-independent project. They exist on the internet, but nobody can find them unless you send the link. A personal portfolio site ties your work together in one place and tells people who you are.

This matters more than you think:

  • Branching: A portfolio with real data science projects signals technical competence in a way that a bullet on a resume cannot.
  • Graduate school: Admissions committees want evidence you can do the work. A live project portfolio is that evidence.
  • Jobs (post-Army or internships): Hiring managers in tech and analytics look at portfolios. A deployed website with working dashboards puts you ahead of candidates who only have a PDF resume.
  • Your own development: Building a site forces you to curate your work and articulate what you have learned.

Today you build and deploy a personal website using Quarto and Netlify.

How It Works

A Quarto website is just a collection of .qmd files plus a _quarto.yml configuration file. You already know Quarto. This is the same tool — just pointed at a different output format.

Key Files

File Purpose
_quarto.yml Site configuration: title, navigation, theme, output format
index.qmd Your landing page — first thing visitors see
about.qmd Bio, background, interests, photo
projects.qmd Links to your NYC 311 dashboard, semi-independent project, and anything else you want to showcase

You can add more pages as you build more projects throughout the semester.

75-Minute Outline

Time Activity
0–5 min Overview. Why portfolios matter. What we are building today.
5–15 min Scaffold the site. Create a new RStudio project (or a new folder in your existing GitHub repo). Set up _quarto.yml, index.qmd, about.qmd, and projects.qmd. Render locally to confirm it works.
15–30 min Write your content. Fill in your landing page with a short intro. Write your about page — who you are, what you study, what interests you. Add a photo if you have one. On the projects page, link to your two deployed dashboards with brief descriptions.
30–40 min Customize. Pick a Quarto theme (cosmo, darkly, flatly, etc.). Adjust colors if you want. Add a navbar. Make it yours.
40–55 min Deploy to Netlify. Push your site to a GitHub repo. Go to netlify.com, connect your GitHub repo, set the build command to quarto render and the publish directory to _site. Deploy. Confirm the live URL works.
55–65 min Test and polish. Click every link. Check the deployed site on your phone. Fix anything that looks off.
65–75 min Share. Post your live URL in the class channel. Browse two classmates’ sites and give them one specific piece of feedback.

Setting Up _quarto.yml

Your site configuration will look something like this:

project:
  type: website

website:
  title: "Your Name"
  navbar:
    left:
      - href: index.qmd
        text: Home
      - href: about.qmd
        text: About
      - href: projects.qmd
        text: Projects

format:
  html:
    theme: cosmo
    css: styles.css

You can change the theme to any Bootswatch theme that Quarto supports.

Deploying to Netlify

Step by step:

  1. Push your site to GitHub. Make sure your repo contains _quarto.yml, your .qmd files, and any images or CSS.
  2. Go to app.netlify.com. Sign up or log in with your GitHub account.
  3. Add new site > Import an existing project > GitHub.
  4. Select your repo.
  5. Build settings:
    • Build command: quarto render
    • Publish directory: _site
  6. Deploy. Netlify builds your site and gives you a URL like random-name-12345.netlify.app.
  7. Rename (optional). In Site settings > Domain management, you can change the subdomain to something like yourname.netlify.app.

Every time you push to GitHub after this, Netlify automatically rebuilds and redeploys your site.

NoteCustom Domain

If you own a domain name (or want to buy one), you can point it at your Netlify site. We will not cover this today, but it is straightforward and Netlify has good documentation. Something to consider down the road.

Tips

  • Keep it simple. A clean, functional site with real projects beats a flashy site with no substance.
  • Write like a human. Your about page should sound like you, not like a LinkedIn profile generated by AI.
  • Link to your code. Include GitHub repo links alongside your deployed dashboards. Employers and admissions committees want to see your code, not just the output.
  • Update it. This site should grow with you. Add projects from this course and others as you complete them.
TipAI Assist

Use AI to help with YAML configuration issues, CSS tweaks, and Netlify troubleshooting. These are exactly the kinds of fiddly problems where AI saves you time. But write your own bio and project descriptions — that is your voice, not AI’s.

Deliverable

By the end of class, submit:

  • Live Netlify URL for your personal website
  • Site must include:
    • A landing page with your name and a brief intro
    • An about page
    • A projects page linking to your NYC 311 dashboard and semi-independent project dashboard
  • GitHub repo linked and pushed

This is the beginning of your professional portfolio. You will keep adding to it.
