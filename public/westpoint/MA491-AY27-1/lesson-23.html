Lesson 23: Guided Example — Spatial Data & Mapping

Overview

Data often has a “where.” Installations, incidents, supply routes, population centers — location matters. Spatial data adds a geometry column to a data frame, letting you filter, join, and plot data on a map just as naturally as you work with numbers and text.

This lesson introduces the sf package for spatial data, static maps with ggplot2, interactive maps with leaflet, and spatial joins. You will build both choropleth maps and point maps.

Objectives

NoteBy the end of this lesson, you will be able to:
  • Explain how sf represents spatial data as data frames with geometry
  • Read shapefiles and other spatial formats
  • Plot spatial data with ggplot2 + geom_sf()
  • Transform coordinate reference systems
  • Build interactive maps with leaflet
  • Perform spatial joins to combine datasets by location

Spatial Data as Data Frames

The sf (simple features) package stores spatial data as regular data frames with one special column: geometry. Every row is a feature (a point, line, or polygon) with attributes and a location.

library(tidyverse)
library(sf)

# Built-in example: North Carolina counties
nc <- st_read(system.file("shape/nc.shp", package = "sf"), quiet = TRUE)

# It's a data frame!
class(nc)
glimpse(nc)
TipKey Insight

Because sf objects are data frames, you can use all your dplyr verbs — filter(), mutate(), group_by(), summarize() — and the geometry column comes along for the ride.

Plotting with ggplot2 + geom_sf()

# Basic map
ggplot(nc) +
  geom_sf() +
  theme_minimal() +
  labs(title = "North Carolina Counties")
# Choropleth: color by a variable
ggplot(nc) +
  geom_sf(aes(fill = BIR74)) +
  scale_fill_viridis_c() +
  theme_minimal() +
  labs(title = "Births in 1974 by County",
       fill = "Births")
NoteChoropleth Maps

A choropleth colors regions by a numeric variable. Use scale_fill_viridis_c() for continuous data or scale_fill_brewer() for categorical data. Always include a legend that explains what the color means.

Reading Shapefiles

Shapefiles are the most common format for spatial data. They come as a set of files (.shp, .shx, .dbf, .prj). You can also read GeoJSON, KML, and other formats.

# Read a shapefile from disk
# my_data <- st_read("path/to/file.shp")

# Read a GeoJSON file
# my_data <- st_read("path/to/file.geojson")

# Download US states from the tigris package
library(tigris)
options(tigris_use_cache = TRUE)

states <- states(cb = TRUE, resolution = "20m")

ggplot(states |> filter(!STUSPS %in% c("HI", "AK", "PR", "GU", "VI", "AS", "MP"))) +
  geom_sf() +
  theme_void() +
  labs(title = "Continental United States")

Coordinate Reference Systems (CRS)

Every spatial dataset has a CRS that defines how coordinates map to locations on Earth. You need all your layers in the same CRS before combining them.

# Check the CRS
st_crs(nc)

# Common EPSG codes
# 4326 = WGS 84 (GPS coordinates, lat/lon) -- most common for web maps
# 3857 = Web Mercator (what Google Maps uses)
# 32618 = UTM Zone 18N (good for eastern US in meters)

# Transform to a different CRS
nc_utm <- st_transform(nc, crs = 32617)
st_crs(nc_utm)
WarningCRS Mismatch

If you try to plot or join two spatial datasets with different CRS, you will get errors or wrong results. Always check with st_crs() and transform with st_transform() before combining data.

Interactive Maps with leaflet

Static maps are good for reports. Interactive maps are good for exploration.

library(leaflet)

# Simple marker map
leaflet() |>
  addTiles() |>
  addMarkers(lng = -73.9576, lat = 41.3915,
             popup = "West Point, NY")

Point Maps

# Multiple points with circle markers
bases <- tibble(
  name = c("West Point", "Fort Liberty", "Fort Moore", "Fort Cavazos"),
  lat  = c(41.3915, 35.1392, 32.3840, 31.1350),
  lng  = c(-73.9576, -79.0064, -84.9564, -97.7753)
)

leaflet(bases) |>
  addTiles() |>
  addCircleMarkers(
    lng = ~lng, lat = ~lat,
    radius = 8,
    color = "darkred",
    fillOpacity = 0.7,
    popup = ~name
  )

Choropleth with Leaflet

# Transform to WGS84 for leaflet (it requires EPSG:4326)
nc_wgs <- st_transform(nc, crs = 4326)

# Color palette
pal <- colorNumeric("YlOrRd", domain = nc_wgs$BIR74)

leaflet(nc_wgs) |>
  addTiles() |>
  addPolygons(
    fillColor = ~pal(BIR74),
    fillOpacity = 0.7,
    weight = 1,
    color = "white",
    popup = ~paste(NAME, "<br>Births:", BIR74)
  ) |>
  addLegend(pal = pal, values = ~BIR74, title = "Births (1974)")

Spatial Joins

A spatial join combines two datasets based on where features are, not a shared key column.

# Create some point data
set.seed(491)
points <- tibble(
  id = 1:20,
  lat = runif(20, min = 34, max = 36.6),
  lng = runif(20, min = -84.3, max = -75.5),
  value = rnorm(20, mean = 50, sd = 10)
) |>
  st_as_sf(coords = c("lng", "lat"), crs = 4326)

# Spatial join: which county is each point in?
points_with_county <- st_join(points, nc_wgs)

points_with_county |>
  select(id, value, NAME) |>
  head(10)
# Aggregate: average value per county
county_avg <- points_with_county |>
  st_drop_geometry() |>
  group_by(NAME) |>
  summarize(avg_value = mean(value), n_points = n())

county_avg
TipWhen to Use Spatial Joins

Use st_join() when you need to ask “which polygon does this point fall in?” or “which points are within this region?” This replaces manual geocoding or lookup tables.

Military Applications

NoteWhere Spatial Analysis Matters
  • Installation analysis: Map facilities, ranges, and training areas; overlay with environmental or demographic data
  • Incident mapping: Plot IED locations, accident sites, or crime incidents to identify clusters
  • Logistics: Route analysis, supply point placement, coverage modeling
  • Force laydown: Visualize unit locations, movement corridors, and area of operations
  • Terrain analysis: Combine elevation, land cover, and road networks for mission planning

Any data with latitude/longitude or region identifiers can become spatial data.
