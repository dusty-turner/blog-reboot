Lesson 30: Capstone Presentations

Overview

This is it. Each cadet presents their end-to-end data science project to the class. You’ll demonstrate everything you’ve built over the semester: from acquiring data to delivering a complete analysis with a Shiny app. Present with confidence – you’ve earned it.

Presentation Format

NoteRequirements
  • 10-minute talk followed by 5-minute Q&A
  • Demo your Shiny app live
  • Communicate results to a non-technical audience

Presentation Order

To be determined.

Evaluation Criteria

Criterion Weight Description
Question & Motivation 15% Is the question clear, relevant, and well-motivated?
Data & Methods 20% Appropriate data acquisition, cleaning, and analytical methods?
Results & Interpretation 25% Are results clearly presented and correctly interpreted?
Shiny App 15% Is the app functional, interactive, and informative?
Communication 15% Clear delivery, good visuals, appropriate for audience?
Q&A 10% Thoughtful responses to questions?

Final Submissions

ImportantDue Today

Submit the following:

  1. GitHub repository URL – clean, documented, reproducible
  2. Rendered Quarto report (HTML or PDF)
  3. Shiny app – deployed to shinyapps.io or confirmed working on the Pi
  4. Reflection memo (1 page): What did you learn? What would you do differently? What skill from this course will you use most?

Audience Expectations

When you’re not presenting:

  • Be an engaged, respectful audience
  • Ask at least one question during the Q&A
  • Take notes – you might learn a technique you can use in your own work
TipGood Questions to Ask
  • “What surprised you most about your results?”
  • “If you had more time, what would you add?”
  • “How did you handle [specific data issue]?”
  • “What was the hardest part of the project?”

Course Wrap-Up

You started the semester with a box of parts. Now you can:

  • Set up computing infrastructure from scratch
  • Write R code for data wrangling, visualization, and modeling
  • Build interactive web applications with Shiny
  • Track your work with Git and communicate results with Quarto
  • Acquire data from files, APIs, and the web
  • Apply regression, classification, text analysis, spatial analysis, and time series methods
  • Scope a project, manage your own workflow, and present results to a real audience
  • Use AI tools responsibly as coding assistants

That’s cradle-to-grave data science. From a Raspberry Pi to a capstone presentation. Well done.
