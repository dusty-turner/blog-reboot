Lesson 5: Data Exploration & Dashboards

Overview

You know how to get data from files, databases, and APIs. Now what? Before you model anything, you explore. Today you’ll work through a complete exploratory data analysis (EDA) workflow, then turn your findings into an interactive dashboard that someone else can use. You’ll use AI tools to help you build it faster.

Objectives

NoteBy the end of this lesson, you will be able to:
  • Perform a structured EDA: data types, missing values, summary stats, distributions, correlations
  • Create informative visualizations: histograms, scatter plots, box plots, bar charts
  • Explain what makes a good dashboard
  • Build an interactive dashboard with at least 2 inputs and 2 reactive outputs
  • Use AI tools (Claude, Copilot) effectively to accelerate dashboard development

Before Class

  • Make sure shiny is installed on your Pi:
install.packages("shiny")
  • Have the tidyverse loaded and working

Part 1: EDA Workflow (20 min)

We’ll use the mpg dataset (built into ggplot2) so there’s zero friction — the data is already on your machine.

Step 1: Look at the data

library(tidyverse)

# What do we have?
glimpse(mpg)
head(mpg)
dim(mpg)  # 234 rows, 11 columns

Step 2: Check data types and missing values

# Data types
sapply(mpg, class)

# Missing values per column
colSums(is.na(mpg))

# Any duplicates?
sum(duplicated(mpg))
TipEDA Checklist

Every time you get a new dataset, ask these questions in order:

  1. How many rows and columns?
  2. What are the data types?
  3. Are there missing values? Where?
  4. What are the distributions of numeric variables?
  5. Are there outliers?
  6. What are the relationships between variables?

Step 3: Summary statistics

# Numeric summaries
summary(mpg |> select(where(is.numeric)))

# Categorical counts
mpg |> count(manufacturer, sort = TRUE)
mpg |> count(class, sort = TRUE)
mpg |> count(drv)

Step 4: Distributions

# Histogram — highway MPG distribution
ggplot(mpg, aes(x = hwy)) +
  geom_histogram(binwidth = 2, fill = "steelblue", color = "white") +
  labs(title = "Distribution of Highway MPG", x = "Highway MPG", y = "Count") +
  theme_minimal()

# Box plot — MPG by vehicle class
ggplot(mpg, aes(x = class, y = hwy, fill = class)) +
  geom_boxplot(show.legend = FALSE) +
  labs(title = "Highway MPG by Vehicle Class", x = "Class", y = "Highway MPG") +
  theme_minimal()

Step 5: Relationships

# Scatter plot — engine size vs. highway MPG
ggplot(mpg, aes(x = displ, y = hwy, color = drv)) +
  geom_point(size = 2) +
  labs(title = "Engine Displacement vs. Highway MPG",
       x = "Engine Displacement (L)", y = "Highway MPG", color = "Drive") +
  theme_minimal()

# Bar chart — count by manufacturer
ggplot(mpg, aes(x = fct_infreq(manufacturer))) +
  geom_bar(fill = "steelblue") +
  labs(title = "Vehicles by Manufacturer", x = "", y = "Count") +
  theme_minimal() +
  theme(axis.text.x = element_text(angle = 45, hjust = 1))

# Correlation matrix for numeric columns
mpg |>
  select(where(is.numeric)) |>
  cor() |>
  round(2)

Part 2: What Makes a Good Dashboard? (10 min)

A dashboard is a single screen that gives someone the key information they need, with the ability to explore further.

Good dashboards:

  • Answer a question — “How does fuel economy vary across vehicle types?”
  • Start with the big picture, then allow drill-down
  • Use the right chart for the data — bar charts for categories, scatter plots for relationships, histograms for distributions
  • Limit clutter — 3-5 well-chosen visuals beat 15 confusing ones
  • Have clear labels and titles — if someone has to ask what a chart shows, it needs a better title

Bad dashboards:

  • Throw every variable onto the page without purpose
  • Use pie charts for everything
  • Have no titles, no axis labels, tiny fonts
  • Require a 10-minute explanation to understand
NoteThink Like the User

Before you build a dashboard, ask: Who will use this? What decision are they making? What do they need to see first?

Part 3: Build a Shiny Dashboard (35 min)

Shiny turns your R code into an interactive web application. It has two parts:

  • UI — what the user sees (inputs, outputs, layout)
  • Server — what happens behind the scenes (data processing, plots)

Starter template

Create a new file ~/ma491/scripts/lesson5_dashboard.R:

library(shiny)
library(tidyverse)

ui <- fluidPage(
  titlePanel("MPG Explorer Dashboard"),

  sidebarLayout(
    sidebarPanel(
      # Input 1: Select vehicle class
      selectInput("class_filter", "Vehicle Class:",
                  choices = c("All", sort(unique(mpg$class))),
                  selected = "All"),

      # Input 2: Choose x-axis variable
      selectInput("x_var", "X-Axis Variable:",
                  choices = c("displ", "cyl", "cty", "hwy", "year"),
                  selected = "displ")
    ),

    mainPanel(
      # Output 1: Scatter plot
      plotOutput("scatter_plot"),

      # Output 2: Summary table
      tableOutput("summary_table")
    )
  )
)

server <- function(input, output) {

  # Reactive data: filter based on user selection
  filtered_data <- reactive({
    if (input$class_filter == "All") {
      mpg
    } else {
      mpg |> filter(class == input$class_filter)
    }
  })

  # Output 1: Scatter plot
  output$scatter_plot <- renderPlot({
    ggplot(filtered_data(), aes(x = .data[[input$x_var]], y = hwy, color = drv)) +
      geom_point(size = 3, alpha = 0.7) +
      labs(title = paste("Highway MPG vs.", input$x_var),
           x = input$x_var, y = "Highway MPG", color = "Drive Type") +
      theme_minimal(base_size = 14)
  })

  # Output 2: Summary table
  output$summary_table <- renderTable({
    filtered_data() |>
      group_by(drv) |>
      summarize(
        Count = n(),
        `Mean MPG (Hwy)` = round(mean(hwy), 1),
        `Mean MPG (City)` = round(mean(cty), 1),
        `Mean Displacement` = round(mean(displ), 1),
        .groups = "drop"
      )
  })
}

shinyApp(ui = ui, server = server)

Run it

In RStudio, open the file and click Run App, or run from the console:

shiny::runApp("~/ma491/scripts/lesson5_dashboard.R")

The app will open in your browser. Try changing the inputs and watch the outputs update.

WarningShiny on a Pi

Shiny runs a local web server. When you run the app from RStudio Server, it’s accessible at the same IP address on a different port. If it doesn’t open automatically, check the console for the URL (something like http://0.0.0.0:XXXX).

Part 4: Extend with AI Assistance (included in Part 3 time)

Now make your dashboard better. Use Claude or GitHub Copilot to help you add features. Here’s the workflow:

  1. Describe what you want in plain English: “Add a histogram below the scatter plot that shows the distribution of the selected x-axis variable, colored by drive type”
  2. Read the AI’s code — don’t just paste blindly
  3. Test it — run the app and verify it works
  4. Iterate — if it’s not right, describe what’s wrong and ask for a fix
TipGood Prompts for Dashboard Building

Try asking your AI assistant things like:

  • “Add a slider input to filter by year range”
  • “Add a download button that exports the filtered data as CSV”
  • “Change the layout to use tabs instead of a single page”
  • “Add a third plot: a box plot of highway MPG by manufacturer for the filtered data”

The goal is to get your dashboard to at least 2 inputs and 2 outputs, but push further if you have time.
