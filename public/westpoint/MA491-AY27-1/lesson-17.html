Lesson 17: Shiny — Interactive Apps

Overview

Static reports are great for communicating results, but sometimes you want your audience to explore the data themselves. Shiny is an R package that turns your analysis into an interactive web application — no HTML, CSS, or JavaScript required. Users can adjust inputs (sliders, dropdowns, date ranges) and watch plots, tables, and text update instantly.

This lesson covers the fundamentals of building, running, and deploying a Shiny app, including hosting on your Raspberry Pi.

Objectives

NoteBy the end of this lesson, you will be able to:
  • Explain Shiny’s UI + Server architecture
  • Build a basic Shiny app with inputs and outputs
  • Use common input widgets and output renderers
  • Understand reactivity: reactive(), observe(), observeEvent()
  • Create modern layouts with bslib
  • Deploy an app to shinyapps.io
  • Host a Shiny app on your Raspberry Pi

What is Shiny?

A Shiny app has two parts:

  1. UI (User Interface) – defines what the user sees: inputs (sliders, buttons, dropdowns) and outputs (plots, tables, text)
  2. Server – defines the logic: how inputs are transformed into outputs
library(shiny)

# UI: what the user sees
ui <- fluidPage(
  titlePanel("My First Shiny App"),
  sidebarLayout(
    sidebarPanel(
      sliderInput("bins", "Number of bins:", min = 5, max = 50, value = 30)
    ),
    mainPanel(
      plotOutput("histogram")
    )
  )
)

# Server: the logic
server <- function(input, output) {
  output$histogram <- renderPlot({
    hist(mtcars$mpg, breaks = input$bins,
         col = "steelblue", border = "white",
         main = "MPG Distribution",
         xlab = "Miles per Gallon")
  })
}

# Run the app
shinyApp(ui = ui, server = server)
TipRunning a Shiny App

Save the code as app.R in its own directory, then run it with shiny::runApp() or click “Run App” in RStudio. The app opens in your browser. Press Ctrl+C in the console (or close the browser window) to stop it.

Common Inputs

Inputs let the user control the app. Each input has an inputId that you reference in the server as input$inputId.

# Slider
sliderInput("n", "Sample size:", min = 10, max = 1000, value = 100)

# Dropdown
selectInput("var", "Variable:", choices = names(mtcars))

# Checkbox
checkboxInput("show_line", "Show trend line?", value = TRUE)

# Radio buttons
radioButtons("color", "Color scheme:",
             choices = c("Blue" = "steelblue", "Red" = "tomato", "Green" = "forestgreen"))

# Text input
textInput("title", "Plot title:", value = "My Plot")

# Date range
dateRangeInput("dates", "Date range:",
               start = "2024-01-01", end = "2024-12-31")

# Numeric input
numericInput("threshold", "Threshold:", value = 20, min = 0, max = 100)

# Action button
actionButton("go", "Run Analysis")

Common Outputs

Outputs display results. Each output function in the UI has a matching render function in the server.

UI function Server function What it shows
plotOutput("id") renderPlot({...}) A plot
tableOutput("id") renderTable({...}) A static table
dataTableOutput("id") renderDataTable({...}) An interactive table
textOutput("id") renderText({...}) Text
verbatimTextOutput("id") renderPrint({...}) Console-style output 
server <- function(input, output) {

  output$my_plot <- renderPlot({
    ggplot(mtcars, aes(x = wt, y = mpg)) +
      geom_point() +
      labs(title = input$title) +
      theme_minimal()
  })

  output$my_table <- renderTable({
    mtcars |>
      group_by(cyl) |>
      summarize(avg_mpg = mean(mpg), .groups = "drop")
  })

  output$my_text <- renderText({
    paste("You selected:", input$var)
  })

  output$my_datatable <- renderDataTable({
    mtcars
  })
}

Reactivity

Reactivity is what makes Shiny work. When an input changes, any output that depends on it automatically re-runs.

reactive() — Create a Reactive Expression

Use reactive() when multiple outputs need the same computed value. It caches the result so it’s only computed once per input change.

server <- function(input, output) {

  # Compute the filtered data once, use it in multiple outputs
  filtered_data <- reactive({
    mtcars |> filter(cyl == input$n_cyl)
  })

  output$plot <- renderPlot({
    ggplot(filtered_data(), aes(x = wt, y = mpg)) +
      geom_point() +
      theme_minimal()
  })

  output$summary <- renderTable({
    summary(filtered_data()$mpg) |> as.data.frame()
  })
}
WarningDon’t Forget the Parentheses

reactive() creates a function. You must call it with parentheses: filtered_data(), not filtered_data.

observe() and observeEvent()

Use these for side effects (things that don’t produce output, like printing to the console or updating an input).

server <- function(input, output) {

  # Runs every time input$var changes
  observe({
    message("User selected: ", input$var)
  })

  # Runs only when the button is clicked
  observeEvent(input$go, {
    message("Button clicked! Running analysis...")
  })
}

Modern Layouts with bslib

The bslib package provides modern, Bootstrap-based layouts that look professional with minimal effort.

library(shiny)
library(bslib)

ui <- page_sidebar(
  title = "mtcars Explorer",
  sidebar = sidebar(
    selectInput("x_var", "X axis:", choices = names(mtcars), selected = "wt"),
    selectInput("y_var", "Y axis:", choices = names(mtcars), selected = "mpg"),
    checkboxInput("smooth", "Add trend line?", value = TRUE)
  ),
  card(
    card_header("Scatter Plot"),
    plotOutput("scatter")
  ),
  card(
    card_header("Summary Statistics"),
    tableOutput("stats")
  )
)

server <- function(input, output) {
  output$scatter <- renderPlot({
    p <- ggplot(mtcars, aes(x = .data[[input$x_var]], y = .data[[input$y_var]])) +
      geom_point(size = 3) +
      theme_minimal()

    if (input$smooth) {
      p <- p + geom_smooth(method = "lm", se = TRUE)
    }
    p
  })

  output$stats <- renderTable({
    mtcars |>
      summarize(
        across(c(all_of(input$x_var), all_of(input$y_var)),
               list(mean = mean, sd = sd, min = min, max = max))
      )
  })
}

shinyApp(ui, server)

Deploying to shinyapps.io

shinyapps.io is the easiest way to share your app with the world. Free tier gives you 5 apps and 25 active hours per month.

# One-time setup
install.packages("rsconnect")
rsconnect::setAccountInfo(
  name = "your-account",
  token = "your-token",
  secret = "your-secret"
)

# Deploy (from the directory containing app.R)
rsconnect::deployApp()
NoteWhat Gets Deployed

shinyapps.io uploads everything in your app directory: app.R, data files, helper scripts. Make sure your data files are in the same directory (or a subdirectory) as app.R. Absolute file paths will not work on the server.

Hosting on Your Raspberry Pi

Your Pi can serve Shiny apps on your local network using Shiny Server.

# Install Shiny Server on the Pi (one-time setup)
sudo apt-get install gdebi-core
wget https://download3.rstudio.org/ubuntu-18.04/aarch64/shiny-server-1.5.21.1012-aarch64.deb
sudo gdebi shiny-server-1.5.21.1012-aarch64.deb

# Your apps go here
sudo cp -r my_app/ /srv/shiny-server/my_app/

# Access the app from any device on the same network
# http://your-pi-ip:3838/my_app/
TipPi Hosting Advantage

With Shiny Server on your Pi, your app runs 24/7 on hardware you control. Anyone on the West Point network can access it through a browser. No shinyapps.io limits, no cloud costs.

NoteTry This

Build a Shiny app that lets users explore the mtcars dataset:

  1. Inputs: A dropdown to select the X variable, a dropdown to select the Y variable, a checkbox to toggle a trend line, and a slider to filter by horsepower range.

  2. Outputs: A scatter plot that updates based on the inputs, a summary table showing mean, median, and sd for both selected variables, and a text line reporting the correlation between the two variables.

  3. Layout: Use bslib::page_sidebar() with cards for each output.

  4. Bonus: Add an action button that downloads the filtered data as a CSV.

Save it as app.R in its own directory and run it.
