Lesson 9: Data Types Deep Dive – Dates, Strings, Factors, NAs

Overview

You can write perfect dplyr pipelines and ggplot charts, and they will still break when your dates are stored as strings, your categories are in the wrong order, or a single NA silently poisons your entire calculation. This lesson is about the data types that cause 80% of beginner pain: dates, strings, factors, and missing values.

None of this is glamorous. All of it is essential. The difference between a data scientist who can handle real data and one who can only work with textbook examples is usually comfort with these topics.

Objectives

NoteBy the end of this lesson, you will be able to:
  • Parse and manipulate dates with lubridate
  • Detect, extract, and replace patterns in strings with stringr
  • Control factor ordering with forcats for clean plots and models
  • Handle missing values (NA) without letting them silently wreck your results
  • Coerce between types: as.numeric(), as.character(), as.Date(), parse_number()
  • Recognize and fix common data type gotchas

Setup

library(tidyverse)
library(lubridate)   # ships with tidyverse but good to be explicit

Dates with lubridate

Parsing Dates

lubridate figures out the date format from the order of components. You just tell it what order to expect.

# Year-month-day
ymd("2025-03-15")

# Month-day-year
mdy("03/15/2025")
mdy("March 15, 2025")

# Day-month-year
dmy("15-03-2025")

# With times
ymd_hms("2025-03-15 14:30:00")
mdy_hm("03/15/2025 2:30 PM")

Extracting Components

d <- ymd("2025-03-15")

year(d)          # 2025
month(d)         # 3
month(d, label = TRUE)  # Mar
day(d)           # 15
wday(d, label = TRUE)   # Sat
week(d)          # 11

Date Arithmetic

# Today and differences
today()
now()

# How many days between two dates?
ymd("2025-05-22") - ymd("2025-01-13")

# Add durations
today() + days(30)
today() + months(6)
today() + years(1)

# Floor/ceiling to periods
floor_date(today(), "month")     # first day of this month
ceiling_date(today(), "week")    # start of next week

Common Date Gotchas

WarningWatch out for these
  • "12/31/2024" vs "2024-12-31" – know your format before parsing
  • Excel serial dates (e.g., 45658) – use as.Date(45658, origin = "1899-12-30")
  • Time zones – lubridate defaults to UTC; use tz = "America/New_York" when it matters
  • "03/04/2025" – is that March 4 or April 3? Context matters. Be explicit with mdy() vs dmy().

Strings with stringr

All stringr functions start with str_ so they autocomplete nicely.

Detection and Matching

fruits <- c("apple", "banana", "pineapple", "grape", "grapefruit")

str_detect(fruits, "apple")      # TRUE FALSE TRUE FALSE FALSE
str_starts(fruits, "grape")      # FALSE FALSE FALSE TRUE TRUE
str_count(fruits, "p")           # 2 0 3 1 0

# Use in a dplyr filter
starwars |>
  filter(str_detect(name, "Skywalker"))

Extraction and Replacement

address <- "West Point, NY 10996"

str_extract(address, "\\d{5}")               # "10996"
str_replace(address, "NY", "New York")        # "West Point, New York 10996"
str_sub(address, 1, 10)                       # "West Point"

# Remove whitespace
str_trim("  hello  ")                          # "hello"
str_squish("too   many   spaces")              # "too many spaces"

Splitting and Combining

str_split("one,two,three", ",")               # list: c("one", "two", "three")

str_c("Hello", "World", sep = " ")             # "Hello World"
str_glue("Today is {today()}")                 # "Today is 2025-03-26"

Regex Basics

Regular expressions (regex) are patterns for matching text. You do not need to master them – just know the basics.

Pattern Matches Example
\\d Any digit "\\d{5}" matches zip codes
\\w Any word character "\\w+" matches words
. Any character "b.t" matches “bat”, “bit”, “but”
^ Start of string "^The" matches strings starting with “The”
$ End of string "ing$" matches strings ending in “ing”
[A-Z] Character class "[A-Z]{2}" matches two uppercase letters
+ One or more "\\d+" matches one or more digits
* Zero or more "\\d*" matches zero or more digits
TipRegex help

When you need a complex regex, ask AI. Explain what you want to match in plain English, and it will write the pattern. Then test it with str_view().

Factors with forcats

Factors are R’s way of representing categorical data. They look like strings but have an underlying integer representation and a defined set of levels (allowed values in a fixed order).

Why Factors Matter

  • ggplot2 orders bars and legends by factor levels
  • Statistical models treat factors as categorical predictors with dummy coding
  • Misspelled categories are caught as invalid levels

Creating and Reordering Factors

# Default: alphabetical order
sizes <- factor(c("Small", "Medium", "Large", "Small", "Large"))
levels(sizes)   # "Large" "Medium" "Small" -- alphabetical, not logical

# Fix with fct_relevel
sizes <- fct_relevel(sizes, "Small", "Medium", "Large")
levels(sizes)   # "Small" "Medium" "Large" -- now correct

Common forcats Functions

# Reorder by another variable (great for plots)
mtcars |>
  mutate(car = rownames(mtcars),
         car = fct_reorder(car, mpg)) |>
  ggplot(aes(x = mpg, y = car)) +
  geom_col() +
  theme_minimal()

# Lump rare categories together
starwars |>
  mutate(species = fct_lump(species, n = 5)) |>
  count(species, sort = TRUE)

# Order by frequency
starwars |>
  mutate(eye_color = fct_infreq(eye_color)) |>
  ggplot(aes(x = eye_color)) +
  geom_bar() +
  coord_flip()

# Reverse factor order
starwars |>
  mutate(eye_color = fct_rev(fct_infreq(eye_color))) |>
  ggplot(aes(x = eye_color)) +
  geom_bar() +
  coord_flip()
WarningFactor gotcha in plots

If your bar chart categories appear in an unhelpful order, the fix is almost always fct_reorder() or fct_infreq(). Resist the urge to manually reorder your data frame rows – fix the factor levels instead.

Missing Values (NA)

NA means “I don’t know.” It is not zero. It is not an empty string. It is the absence of a value, and it propagates through calculations.

NA Propagation

x <- c(1, 2, NA, 4, 5)

mean(x)          # NA -- the answer is "I don't know"
mean(x, na.rm = TRUE)   # 3 -- ignore missing values

sum(x)           # NA
sum(x, na.rm = TRUE)    # 12

Detecting NAs

is.na(x)                    # FALSE FALSE TRUE FALSE FALSE

# Count NAs per column
colSums(is.na(airquality))

# Which rows are complete?
complete.cases(airquality) |> sum()

Handling NAs in Data Frames

# Drop rows with any NA
airquality |> drop_na()

# Drop rows with NA in specific columns
airquality |> drop_na(Ozone)

# Replace NA with a value
airquality |>
  mutate(Ozone = replace_na(Ozone, median(Ozone, na.rm = TRUE)))

# Replace NA with a value using coalesce (takes first non-NA)
airquality |>
  mutate(Ozone = coalesce(Ozone, 0))

Understanding Why Data is Missing

NoteThree types of missingness
  1. MCAR (Missing Completely at Random) – the missingness has nothing to do with the data. Safe to drop.
  2. MAR (Missing at Random) – the missingness is related to other observed variables. Imputation helps.
  3. MNAR (Missing Not at Random) – the missingness is related to the missing value itself (e.g., people with high income refuse to report income). Hardest to handle.

Before you drop or impute, ask: why is this value missing? The answer changes your strategy.

Type Coercion

Sometimes data arrives as the wrong type. Fix it explicitly.

# String to number
as.numeric("42")          # 42
as.numeric("forty-two")   # NA (with a warning)

# Number to string
as.character(42)           # "42"

# String to date
as.Date("2025-03-15")     # works if format is YYYY-MM-DD

# Parse messy numbers (readr)
parse_number("$1,234.56")  # 1234.56
parse_number("30%")         # 30

# Check types
class(42)           # "numeric"
class("hello")      # "character"
class(TRUE)         # "logical"
class(today())      # "Date"
WarningSilent type problems

CSV files store everything as text. read_csv() guesses column types but sometimes guesses wrong. A column of zip codes like “01234” gets read as the number 1234. Check your types with glimpse() after every file read.

Common Gotchas Summary

Problem Symptom Fix
Date parsed as character Dates sort wrong, arithmetic fails ymd(), mdy(), or as.Date()
Excel serial date (45658) Numbers where dates should be as.Date(x, origin = "1899-12-30")
Bars in wrong order Alphabetical instead of logical fct_relevel() or fct_reorder()
mean() returns NA One missing value in the vector Add na.rm = TRUE
Zip codes lose leading zeros “01234” becomes 1234 Read as character: col_types = cols(zip = col_character())
"TRUE" vs TRUE String comparison fails as.logical()
Factor vs character Model treats variable differently Be intentional: factor() when categorical, character when just text
