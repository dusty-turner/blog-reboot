Lesson 21: Guided Example — Classification (Part 2: Trees, Random Forest, Model Comparison)

Overview

Logistic regression is a strong baseline, but it is not the only classifier. Decision trees are intuitive — they split data using yes/no questions, and you can draw the logic as a flowchart. Random forests combine many trees to improve accuracy at the cost of interpretability.

Today you will fit decision trees and random forests, compare them to logistic regression, and think carefully about when interpretability matters more than accuracy.

Objectives

NoteBy the end of this lesson, you will be able to:
  • Fit and visualize a decision tree
  • Fit a random forest model
  • Compare multiple classifiers on the same test set
  • Explain the accuracy vs interpretability tradeoff
  • Use cross-validation for more robust model evaluation

Setup

We continue with the transmission classification problem from Lesson 20.

library(tidyverse)
library(tidymodels)
library(rpart.plot)

# Data prep (same as Lesson 20)
set.seed(491)

cars <- mtcars |>
  mutate(transmission = factor(am, labels = c("automatic", "manual"))) |>
  select(transmission, wt, hp, cyl, mpg, disp)

car_split <- initial_split(cars, prop = 0.75, strata = transmission)
car_train <- training(car_split)
car_test  <- testing(car_split)

Decision Trees

A decision tree splits the data one variable at a time, choosing the split that best separates the classes.

# Specify and fit
tree_spec <- decision_tree() |>
  set_engine("rpart") |>
  set_mode("classification")

tree_fit <- tree_spec |>
  fit(transmission ~ ., data = car_train)

# View the tree structure
tree_fit

Visualizing the Tree

# Extract the underlying rpart object and plot
rpart.plot(tree_fit$fit, roundint = FALSE, extra = 104)
TipWhy Trees Are Interpretable

You can hand someone a decision tree diagram and they can follow it without knowing any statistics. Each node is a yes/no question. That makes trees excellent for briefing decision-makers who need to understand the logic, not just the answer.

Evaluate the Tree

tree_preds <- predict(tree_fit, car_test) |>
  bind_cols(car_test)

conf_mat(tree_preds, truth = transmission, estimate = .pred_class)
accuracy(tree_preds, truth = transmission, estimate = .pred_class)

Random Forests

A random forest builds hundreds of trees, each on a random subset of the data and a random subset of predictors. The final prediction is a majority vote across all trees.

rf_spec <- rand_forest(trees = 500) |>
  set_engine("ranger") |>
  set_mode("classification")

rf_fit <- rf_spec |>
  fit(transmission ~ ., data = car_train)

rf_fit
NoteWhy ‘Random’?

Two sources of randomness make the forest stronger than any single tree:

  1. Bootstrap sampling: each tree sees a different random sample of rows
  2. Feature sampling: each split considers a random subset of predictors

This prevents all the trees from being the same and reduces overfitting.

Evaluate the Random Forest

rf_preds <- predict(rf_fit, car_test) |>
  bind_cols(car_test)

conf_mat(rf_preds, truth = transmission, estimate = .pred_class)
accuracy(rf_preds, truth = transmission, estimate = .pred_class)

Model Comparison

Let’s compare all three models side by side.

# Refit logistic regression for comparison
log_fit <- logistic_reg() |>
  set_engine("glm") |>
  set_mode("classification") |>
  fit(transmission ~ ., data = car_train)

log_preds <- predict(log_fit, car_test) |>
  bind_cols(car_test)

# Collect accuracy from each
bind_rows(
  accuracy(log_preds,  truth = transmission, estimate = .pred_class) |> mutate(model = "Logistic Regression"),
  accuracy(tree_preds, truth = transmission, estimate = .pred_class) |> mutate(model = "Decision Tree"),
  accuracy(rf_preds,   truth = transmission, estimate = .pred_class) |> mutate(model = "Random Forest")
) |>
  select(model, .metric, .estimate)

Accuracy vs Interpretability

There is no free lunch. More accurate models are often harder to explain.

Model Interpretability Typical Accuracy Best for…
Logistic Regression High Moderate When you need to explain why (inference)
Decision Tree Very High Moderate Briefings, rule-based decisions
Random Forest Low High When prediction accuracy matters most
WarningKnow Your Audience

If you are briefing a battalion commander, a decision tree diagram may be more useful than a random forest with 0.02 higher accuracy. If you are building a system that screens thousands of records, accuracy wins. Context drives the choice.

Cross-Validation

With only 32 rows in mtcars, a single train/test split can be noisy. Cross-validation gives a more robust estimate by splitting the data multiple times.

# Create 5 folds
set.seed(491)
car_folds <- vfold_cv(cars, v = 5, strata = transmission)

# Cross-validate the logistic regression
log_cv <- logistic_reg() |>
  set_engine("glm") |>
  set_mode("classification") |>
  fit_resamples(
    transmission ~ .,
    resamples = car_folds,
    metrics = metric_set(accuracy, precision, recall)
  )

collect_metrics(log_cv)
# Cross-validate the random forest
rf_cv <- rand_forest(trees = 500) |>
  set_engine("ranger") |>
  set_mode("classification") |>
  fit_resamples(
    transmission ~ .,
    resamples = car_folds,
    metrics = metric_set(accuracy, precision, recall)
  )

collect_metrics(rf_cv)
TipHow Cross-Validation Works

With 5-fold CV, the data is split into 5 equal parts. Each fold takes a turn as the test set while the other 4 are training data. The final metric is the average across all 5 folds — much more stable than a single split.

The Full Classification Workflow

  1. Question: Which class does this observation belong to?
  2. Data: Clean, encode the outcome as a factor
  3. Split: Train/test (or cross-validation)
  4. Model(s): Fit one or more classifiers
  5. Evaluate: Confusion matrix, accuracy, precision, recall
  6. Compare: Which model fits the problem best? Consider accuracy and interpretability
  7. Communicate: Present results honestly, including error types and their costs
