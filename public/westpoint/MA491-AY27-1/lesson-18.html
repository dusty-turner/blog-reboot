Lesson 18: Guided Example — Regression (Part 1: Question, Data, EDA)

Overview

This is the first of a two-part guided example walking through a complete regression analysis. We’ll follow the full data science workflow: start with a question, get the data, explore it thoroughly, and build an initial model. Part 2 (next lesson) will cover diagnostics, multiple regression, and interpretation.

The question: How does a car’s weight and engine size relate to its fuel efficiency?

The data: mtcars — 32 automobiles from 1974 Motor Trend magazine. Small, but perfect for learning the workflow.

Objectives

NoteBy the end of this lesson, you will be able to:
  • Articulate a clear analytical question before touching any data
  • Conduct systematic exploratory data analysis (EDA)
  • Identify potential issues like multicollinearity and outliers
  • Fit a simple linear regression model with lm()
  • Extract tidy model results with broom
  • Interpret a slope coefficient in context
  • Visualize the model fit

The Workflow

Every data science project follows some version of this loop:

  1. Question – What are you trying to learn?
  2. Data – Where does it come from? What does it contain?
  3. EDA – What patterns, relationships, and problems exist in the data?
  4. Model – Fit a model that captures the relationship
  5. Diagnose – Does the model meet its assumptions?
  6. Interpret – What do the results mean in context?
  7. Communicate – Tell the story clearly

Today we’ll cover steps 1 through the beginning of step 4. Next lesson finishes the job.

Step 1: The Question

ImportantStart with a Question

Never start by looking at the data. Start by writing down what you want to learn. This keeps you from going down rabbit holes.

Our question: How does a car’s weight and engine size relate to fuel efficiency (MPG)?

This is a good question because:

  • It identifies a clear response variable (MPG)
  • It identifies specific predictors (weight, engine size)
  • It’s answerable with the data we have
  • It has practical relevance (heavier cars cost more to drive)

Step 2: Load and Explore the Data

library(tidyverse)
library(broom)

# Load the data
data(mtcars)

# What do we have?
glimpse(mtcars)
# Summary statistics for all variables
summary(mtcars)
# Focus on the variables we care about
mtcars |>
  select(mpg, wt, hp, disp, cyl) |>
  summary()
NoteKey Variables
Variable Description Units
mpg Fuel efficiency Miles per gallon
wt Weight Thousands of pounds
hp Horsepower Gross horsepower
disp Displacement Cubic inches
cyl Cylinders Count (4, 6, or 8)

Step 3: Exploratory Data Analysis

Distribution of the Response Variable

Before modeling, understand what you’re trying to predict.

# Distribution of MPG
ggplot(mtcars, aes(x = mpg)) +
  geom_histogram(bins = 12, fill = "steelblue", color = "white") +
  geom_vline(xintercept = mean(mtcars$mpg), color = "red", linewidth = 1, linetype = "dashed") +
  labs(title = "Distribution of Fuel Efficiency",
       subtitle = "Red line = mean",
       x = "Miles per Gallon", y = "Count") +
  theme_minimal()
# Quick stats
mtcars |>
  summarize(
    mean_mpg = mean(mpg),
    median_mpg = median(mpg),
    sd_mpg = sd(mpg),
    min_mpg = min(mpg),
    max_mpg = max(mpg)
  )

TipWhat Do You See?

All three scatterplots show a negative relationship: as weight/horsepower/displacement increase, MPG decreases. The relationship with weight looks the most linear. Horsepower and displacement show some curvature.

Identifying Potential Issues

WarningMulticollinearity

Notice that wt, hp, and disp are highly correlated with each other (r > 0.65). This is multicollinearity — the predictors are related to each other, not just to the response. This doesn’t prevent us from fitting a model, but it makes interpreting individual coefficients tricky. We’ll address this in Part 2.

# Check for outliers using boxplots
mtcars |>
  select(mpg, wt, hp, disp) |>
  pivot_longer(everything(), names_to = "variable", values_to = "value") |>
  ggplot(aes(x = variable, y = value)) +
  geom_boxplot(fill = "steelblue", alpha = 0.5) +
  facet_wrap(~variable, scales = "free") +
  labs(title = "Boxplots of Key Variables") +
  theme_minimal()

Step 4: Simple Linear Regression

Let’s start simple: predict MPG from weight alone.

Fitting the Model

# Fit a simple linear regression: mpg = b0 + b1*wt + error
model_simple <- lm(mpg ~ wt, data = mtcars)

# Base R summary
summary(model_simple)

Tidy Output with broom

The broom package converts model output into tidy data frames — much easier to work with.

# Coefficient estimates
tidy(model_simple)

# Model-level statistics (R-squared, F-statistic, etc.)
glance(model_simple)

# Row-level diagnostics (fitted values, residuals, etc.)
augment(model_simple) |> head()

Interpreting the Coefficients

tidy(model_simple)

This gives us two rows:

  • (Intercept): ~37.3 – A car weighing 0 lbs would get 37.3 mpg. (Nonsensical, but it anchors the line.)
  • wt: ~-5.3 – For each additional 1,000 lbs of weight, fuel efficiency decreases by about 5.3 mpg.
NoteInterpreting the Slope

The slope of -5.3 means: holding everything else constant, a car that weighs 1,000 lbs more than another gets about 5.3 fewer miles per gallon. This is a large practical effect. Going from a 2,500 lb car to a 3,500 lb car costs you over 5 mpg.

Model Fit

# R-squared
glance(model_simple)$r.squared

The R-squared is about 0.75, meaning weight alone explains 75% of the variation in fuel efficiency. That’s a strong single-predictor model.

Visualizing the Fit

ggplot(mtcars, aes(x = wt, y = mpg)) +
  geom_point(size = 3) +
  geom_smooth(method = "lm", se = TRUE, color = "steelblue") +
  labs(title = "Simple Linear Regression: MPG ~ Weight",
       subtitle = paste("R-squared =", round(glance(model_simple)$r.squared, 3)),
       x = "Weight (1000 lbs)",
       y = "Miles per Gallon") +
  theme_minimal()
# Color by cylinder count to see if there's structure we're missing
ggplot(mtcars, aes(x = wt, y = mpg, color = factor(cyl))) +
  geom_point(size = 3) +
  geom_smooth(method = "lm", se = FALSE, linetype = "dashed") +
  labs(title = "MPG vs Weight, Colored by Cylinders",
       x = "Weight (1000 lbs)",
       y = "Miles per Gallon",
       color = "Cylinders") +
  theme_minimal()
TipWhat Does the Colored Plot Reveal?

The cylinder groups cluster together. 4-cylinder cars are light and efficient, 8-cylinder cars are heavy and inefficient. Within each group, the weight-MPG relationship still holds but might have a different slope. This suggests we should consider adding cyl to the model — we’ll do that in Part 2.
