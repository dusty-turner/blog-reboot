Lesson 13: Getting Data — Files, APIs, Web Scraping

Overview

Before you can analyze anything, you need data. Sometimes it’s a CSV sitting on your desktop. Sometimes it lives behind an API. Sometimes it’s trapped in an HTML table on a government website. This lesson covers three fundamental approaches to getting data into R: reading local files, querying web APIs, and scraping web pages. You’ll also learn how to use AI tools to discover and evaluate data sources you didn’t know existed.

Objectives

NoteBy the end of this lesson, you will be able to:
  • Read CSV and Excel files into R with readr and readxl
  • Query a REST API using httr2 and parse the JSON response
  • Scrape tables and text from web pages using rvest
  • Articulate the ethical boundaries of web scraping
  • Use AI tools to identify and acquire relevant data sources

Reading CSV Files with readr

The readr package (part of the tidyverse) provides fast, consistent functions for reading delimited files.

library(tidyverse)

# Read a CSV file
weather <- read_csv("data/weather_station.csv")

# read_csv() is smart about types, but you can override
weather <- read_csv("data/weather_station.csv",
  col_types = cols(
    date = col_date(format = "%Y-%m-%d"),
    temp_f = col_double(),
    station_id = col_character()
  )
)

# Quick look
glimpse(weather)
Tipread_csv vs read.csv

Use read_csv() (underscore) from readr, not read.csv() (dot) from base R. The readr version is faster, produces tibbles, and handles column types more intelligently.

Reading Excel Files with readxl

Excel files are everywhere in the Army. The readxl package reads .xls and .xlsx files without requiring Excel to be installed.

library(readxl)

# See what sheets are in the workbook
excel_sheets("data/unit_readiness.xlsx")

# Read a specific sheet
personnel <- read_excel("data/unit_readiness.xlsx", sheet = "Personnel")

# Read a range
summary_data <- read_excel("data/unit_readiness.xlsx",
  sheet = "Summary",
  range = "A1:F50",
  skip = 2
)

glimpse(personnel)
WarningExcel Gotchas
  • Dates in Excel can come in as numbers (e.g., 45292). Use janitor::excel_numeric_to_date() to fix them.
  • Merged cells cause problems. Unmerge them in Excel first, or use readxl’s col_names argument.
  • Hidden sheets still show up in excel_sheets().

REST APIs with httr2

An API (Application Programming Interface) lets you request data from a web service programmatically. Most modern APIs return JSON. The httr2 package makes this straightforward.

library(httr2)

# Example: Open-Meteo weather API (free, no key required)
# Get the forecast for West Point, NY
resp <- request("https://api.open-meteo.com/v1/forecast") |>
  req_url_query(
    latitude = 41.39,
    longitude = -73.95,
    daily = "temperature_2m_max,temperature_2m_min,precipitation_sum",
    temperature_unit = "fahrenheit",
    timezone = "America/New_York",
    forecast_days = 7
  ) |>
  req_perform()

# Check status
resp_status(resp)

# Parse the JSON response
weather_data <- resp_body_json(resp)

# The structure is nested -- explore it
str(weather_data$daily)

# Convert to a tibble
forecast <- tibble(
  date = as.Date(unlist(weather_data$daily$time)),
  high = unlist(weather_data$daily$temperature_2m_max),
  low = unlist(weather_data$daily$temperature_2m_min),
  precip = unlist(weather_data$daily$precipitation_sum)
)

forecast

API Tips

NoteWorking with APIs

API Keys: Many APIs require authentication. Store your key in an environment variable, never in your code:

# In your .Renviron file (run usethis::edit_r_environ()):
# MY_API_KEY=abc123xyz

# In your code:
my_key <- Sys.getenv("MY_API_KEY")

resp <- request("https://api.example.com/data") |>
  req_headers(Authorization = paste("Bearer", my_key)) |>
  req_perform()

Rate Limits: APIs limit how many requests you can make. Be polite:

# Add a delay between requests
resp <- request("https://api.example.com/data") |>
  req_throttle(rate = 10 / 60) |>  # 10 requests per minute
  req_perform()

Caching: Don’t re-download data you already have. Save API responses locally:

# Save the parsed data
write_csv(forecast, "data/forecast_cache.csv")

# Next time, check if the cache exists first
if (file.exists("data/forecast_cache.csv")) {
  forecast <- read_csv("data/forecast_cache.csv")
} else {
  # make the API call
}

Web Scraping with rvest

When data lives on a web page but there’s no API, you can scrape it. The rvest package makes this manageable.

library(rvest)

# Read a web page
page <- read_html("https://en.wikipedia.org/wiki/List_of_U.S._states_by_population")

# Extract all tables from the page
tables <- html_table(page)

# The first table is usually what you want
population <- tables[[1]]
glimpse(population)

# You can also target specific elements using CSS selectors
page <- read_html("https://example.com/data-page")

# Get all paragraph text
paragraphs <- page |>
  html_elements("p") |>
  html_text()

# Get text from elements with a specific class
values <- page |>
  html_elements(".data-value") |>
  html_text()

# Get an attribute (like href from links)
links <- page |>
  html_elements("a") |>
  html_attr("href")

Scraping Ethics

WarningEthical Web Scraping

Web scraping exists in a legal and ethical gray area. Follow these principles:

  1. Check robots.txt – Visit https://example.com/robots.txt before scraping. It tells you what the site allows bots to access.

  2. Read the Terms of Service – Some sites explicitly prohibit scraping. Respect that.

  3. Don’t scrape personal data – Names, emails, phone numbers, and other PII are off limits unless the data is genuinely public and you have a legitimate purpose.

  4. Be gentle – Add delays between requests. Don’t hammer a server with thousands of requests per second.

  5. Prefer APIs – If the site offers an API, use it instead of scraping. It’s more reliable and more respectful.

  6. Cache your results – Download once, analyze many times. Don’t re-scrape data you already have.

As a general rule: if you wouldn’t be comfortable explaining your scraping to the site owner, don’t do it.

Using AI to Find Data Sources

AI tools (ChatGPT, Claude, Copilot) are surprisingly good at helping you find data. Try prompts like:

  • “What public datasets exist about military installation energy usage?”
  • “I need daily weather data for 50 US cities going back 10 years. What APIs or datasets are available?”
  • “Is there a public API for Army recruiting statistics?”
TipAI-Assisted Data Discovery Workflow
  1. Describe your question to an AI tool. Be specific about what variables you need, the time range, and the geographic scope.
  2. Ask for data sources – the AI will suggest datasets, APIs, and government data portals you may not have known about.
  3. Verify the source – AI can hallucinate URLs and dataset names. Always click the link and confirm the data actually exists.
  4. Ask for code – Once you’ve confirmed the source, ask the AI to write the httr2 or read_csv code to pull the data.
  5. Validate the data – Run glimpse(), summary(), and check for obvious problems before trusting it.
