Lesson 12: Exploratory Data Analysis – A Systematic Workflow

Overview

Exploratory Data Analysis (EDA) is where everything from the last five lessons comes together. You have dplyr for wrangling, ggplot2 for visualization, lubridate/stringr/forcats for data types, tidyr for reshaping, and AI as a collaborator. Now you put them all to work on the same dataset in a disciplined, iterative process.

EDA is not a checklist to complete – it is a cycle. You ask a question, wrangle the data to answer it, visualize the result, and that visualization raises a new question. The best analysts run this loop fast and often. By the end of this lesson, you should have a repeatable workflow you can apply to any dataset you encounter for the rest of the course and beyond.

Objectives

NoteBy the end of this lesson, you will be able to:
  • Follow the EDA workflow: question, wrangle, visualize, summarize, communicate
  • Compute and interpret summary statistics (mean, median, sd, IQR, quantiles)
  • Produce by-group summaries with group_by() + summarize() + across()
  • Visualize distributions, correlations, and relationships
  • Assess and handle missing data systematically
  • Use AI to accelerate (but not replace) exploration
  • Run a complete EDA on a dataset from scratch

Setup

library(tidyverse)

The EDA Workflow

1. QUESTION  →  2. WRANGLE  →  3. VISUALIZE  →  4. SUMMARIZE  →  5. COMMUNICATE
      ↑                                                                    |
      └────────────────────── new questions ←──────────────────────────────┘
  1. Question: What are you trying to learn? Start broad, get specific.
  2. Wrangle: Get the data into shape for the question at hand.
  3. Visualize: Plot it. Look for patterns, outliers, surprises.
  4. Summarize: Compute numbers that quantify what you see.
  5. Communicate: Write down what you found. This often raises the next question.

This loop runs many times per analysis. Each pass sharpens your understanding.

Step 1: First Contact with a Dataset

Before you ask any specific questions, get oriented.

# How big is it?
dim(mtcars)

# What are the variables?
glimpse(mtcars)

# Quick summary of every column
summary(mtcars)

# First few rows
head(mtcars)
TipThe 30-second scan

Every time you open a new dataset, run dim(), glimpse(), and summary() before anything else. This takes 30 seconds and prevents 30 minutes of confusion later.

Step 2: Summary Statistics

Overall Summaries

mtcars |>
  summarize(
    mean_mpg = mean(mpg),
    median_mpg = median(mpg),
    sd_mpg = sd(mpg),
    iqr_mpg = IQR(mpg),
    min_mpg = min(mpg),
    max_mpg = max(mpg),
    q25 = quantile(mpg, 0.25),
    q75 = quantile(mpg, 0.75)
  )

By-Group Summaries

# Summary by cylinder count
mtcars |>
  group_by(cyl) |>
  summarize(
    avg_mpg = mean(mpg),
    sd_mpg = sd(mpg),
    avg_hp = mean(hp),
    n = n()
  )

Summarize Across Multiple Columns

across() lets you apply the same summary to many columns at once.

# Mean and SD of all numeric columns, by group
mtcars |>
  group_by(cyl) |>
  summarize(across(c(mpg, hp, wt, disp),
                   list(mean = mean, sd = sd),
                   .names = "{.col}_{.fn}"))
# Apply to all numeric columns
mtcars |>
  group_by(cyl) |>
  summarize(across(where(is.numeric), mean))

Step 3: Distributions

Understanding how each variable is distributed is the foundation of EDA.

Histograms

ggplot(mtcars, aes(x = mpg)) +
  geom_histogram(bins = 12, fill = "steelblue", color = "white") +
  labs(title = "Distribution of MPG") +
  theme_minimal()

Density Plots

ggplot(mtcars, aes(x = mpg, fill = factor(cyl))) +
  geom_density(alpha = 0.5) +
  labs(title = "MPG Distribution by Cylinder Count",
       fill = "Cylinders") +
  theme_minimal()

Boxplots for Comparison

ggplot(mtcars, aes(x = factor(cyl), y = mpg)) +
  geom_boxplot(fill = "lightblue") +
  geom_jitter(width = 0.1, alpha = 0.4) +   # show the actual points
  labs(x = "Cylinders", y = "MPG",
       title = "MPG by Cylinder Count") +
  theme_minimal()
TipAlways add points to boxplots

A boxplot of 5 data points looks the same as a boxplot of 500. Adding geom_jitter() shows you how much data is behind each box.

Step 4: Correlations

Correlation Matrix

# Pairwise correlations
mtcars |>
  select(mpg, wt, hp, disp, drat) |>
  cor() |>
  round(2)

Scatterplot Matrix

# Base R pairs plot
mtcars |>
  select(mpg, wt, hp, disp) |>
  pairs()

GGally for Richer Pair Plots

library(GGally)

mtcars |>
  select(mpg, wt, hp, cyl) |>
  mutate(cyl = factor(cyl)) |>
  ggpairs(aes(color = cyl), upper = list(continuous = "cor")) +
  theme_minimal()
WarningCorrelation is not causation

A strong correlation between two variables does not mean one causes the other. It might. It might not. Weight and MPG are correlated, but adding weight to a car does not directly reduce its MPG – engine size, aerodynamics, and drivetrain are confounders.

Step 5: Missing Data

How Much is Missing?

# Count NAs per column
colSums(is.na(airquality))

# As a proportion
colMeans(is.na(airquality)) |> round(3)

Visualize Missingness

library(naniar)

# Bar chart of missing counts per variable
gg_miss_var(airquality) +
  labs(title = "Missing Values by Variable")

# Heatmap of missingness patterns
vis_miss(airquality) +
  labs(title = "Missingness Pattern")

Strategies for Missing Data

Strategy When to use Code
Drop rows Few NAs, large dataset drop_na()
Drop specific columns Column is mostly NA select(-col)
Replace with median/mean MCAR, want to preserve sample size replace_na()
Flag it Missingness itself might be informative mutate(was_missing = is.na(x))
Investigate first Always Ask why it is missing 
# Replace Ozone NAs with the monthly median
airquality |>
  group_by(Month) |>
  mutate(Ozone = replace_na(Ozone, median(Ozone, na.rm = TRUE))) |>
  ungroup()
WarningDropping data has consequences

If you start with 153 rows and drop_na() leaves you with 111, you just lost 27% of your data. That might be fine, or it might introduce bias if the missing values are not random. Always report how much data you dropped and why.

Using AI for EDA Acceleration

AI is particularly useful during EDA because exploration is open-ended and repetitive. Good prompts:

  • “I have a dataset with these columns: [paste glimpse() output]. Write an EDA starter script that checks distributions, correlations, and missing data.”
  • “Suggest three interesting questions I could explore with this data.”
  • “Write code to check for outliers in the income column using the IQR method.”
  • “Create a faceted plot comparing the distribution of score across region and year.”

Then apply the verification workflow from Lesson 11: read the code, run it, test it, own it.

TipAI + EDA = fast iteration

The EDA loop (question, wrangle, visualize, summarize) runs faster when AI writes the boilerplate and you focus on interpreting the results. Ask the question, let AI write the code, verify the output, form the next question.

Full EDA Example

Here is a compressed example of a full EDA on mtcars. In practice, each step would be more detailed.

# 1. QUESTION: How do car characteristics relate to fuel efficiency?

# 2. WRANGLE: Prepare the data
cars <- mtcars |>
  mutate(
    car = rownames(mtcars),
    cyl = factor(cyl),
    am = factor(am, labels = c("Automatic", "Manual")),
    vs = factor(vs, labels = c("V-shaped", "Straight"))
  )

# 3. VISUALIZE: Key relationships
# MPG by weight, colored by cylinders
ggplot(cars, aes(x = wt, y = mpg, color = cyl)) +
  geom_point(size = 3) +
  geom_smooth(method = "lm", se = FALSE) +
  labs(title = "Heavier cars get worse mileage",
       x = "Weight (1000 lbs)", y = "MPG", color = "Cylinders") +
  theme_minimal()

# MPG by transmission type
ggplot(cars, aes(x = am, y = mpg)) +
  geom_boxplot(fill = "lightblue") +
  geom_jitter(width = 0.1, alpha = 0.4) +
  labs(title = "Manual transmissions tend to get better mileage",
       x = "Transmission", y = "MPG") +
  theme_minimal()

# 4. SUMMARIZE: Key numbers
cars |>
  group_by(cyl, am) |>
  summarize(
    avg_mpg = mean(mpg),
    sd_mpg = sd(mpg),
    avg_wt = mean(wt),
    n = n(),
    .groups = "drop"
  ) |>
  arrange(cyl, am)

# 5. COMMUNICATE: Manual 4-cylinder cars average ~28 MPG.
#    Automatic 8-cylinder cars average ~15 MPG.
#    Weight is the strongest single predictor (r = -0.87).
#    But: confounded -- lighter cars tend to have fewer cylinders
#    and manual transmissions. Need modeling to disentangle.

The EDA Report Skeleton

When you write up an EDA (in Quarto), follow this structure:

  1. Data description: Where did it come from? How many rows/columns? What are the variables?
  2. Missing data: How much? Which variables? What did you do about it?
  3. Univariate summaries: Distribution of each key variable
  4. Bivariate relationships: Correlations, scatterplots, grouped comparisons
  5. Key findings: 3-5 bullet points of what you learned
  6. Next steps: What questions remain? What analysis should come next?
