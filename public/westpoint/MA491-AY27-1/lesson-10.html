Lesson 10: Tidy Data – Pivots, Joins, Reshaping

Overview

Most real-world data is messy. Columns are mislabeled, variables are spread across multiple columns, observations are split across rows, and the dataset you need is actually two datasets that have to be combined. This lesson teaches you the tools to fix all of that.

The core idea is tidy data: each variable is a column, each observation is a row, each value is a cell. When your data is tidy, every tool in the tidyverse works smoothly. When it is not, you fight the tools at every step. Learning to reshape data is the single highest-leverage skill in practical data science.

Objectives

NoteBy the end of this lesson, you will be able to:
  • Explain the tidy data principles
  • Reshape wide data to long with pivot_longer()
  • Reshape long data to wide with pivot_wider()
  • Combine datasets with left_join(), inner_join(), full_join(), and anti_join()
  • Split and combine columns with separate() and unite()
  • Take a messy real-world dataset and make it tidy

Setup

library(tidyverse)

What Is Tidy Data?

Tidy data has three rules:

  1. Each variable is a column
  2. Each observation is a row
  3. Each value is a cell

Most data you encounter violates at least one of these. The fix is usually a pivot.

Example: Messy vs. Tidy

Messy (wide) – values of a variable (year) are spread across column names:

country 1999 2000
Afghanistan 745 2666
Brazil 37737 80488

Tidy (long) – year is now a variable in its own column:

country year cases
Afghanistan 1999 745
Afghanistan 2000 2666
Brazil 1999 37737
Brazil 2000 80488

pivot_longer() – Wide to Long

Use when column names contain data values (like years, months, or categories).

# table4a has columns: country, `1999`, `2000`
table4a

table4a |>
  pivot_longer(
    cols = c(`1999`, `2000`),    # columns to pivot
    names_to = "year",            # new column for old column names
    values_to = "cases"           # new column for the values
  )

More Complex Pivots

# When you have many columns to pivot, use helpers
# Suppose you have: id, score_math, score_reading, score_science
scores <- tibble(
  student = c("Alice", "Bob"),
  score_math = c(85, 92),
  score_reading = c(90, 78),
  score_science = c(88, 95)
)

scores |>
  pivot_longer(
    cols = starts_with("score_"),
    names_to = "subject",
    names_prefix = "score_",       # strip the prefix
    values_to = "score"
  )
# Convert year to integer during the pivot
table4a |>
  pivot_longer(
    cols = c(`1999`, `2000`),
    names_to = "year",
    names_transform = list(year = as.integer),
    values_to = "cases"
  )

pivot_wider() – Long to Wide

Use when an observation is scattered across multiple rows.

# table2 has: country, year, type (cases/population), count
table2

table2 |>
  pivot_wider(
    names_from = type,       # values become column names
    values_from = count      # values fill the new columns
  )

When to Go Wide

  • Creating summary tables for presentation
  • When you need one row per subject (e.g., for modeling)
  • Cross-tabulations
# Create a cross-tabulation
mtcars |>
  count(cyl, gear) |>
  pivot_wider(
    names_from = gear,
    values_from = n,
    values_fill = 0          # fill missing combos with 0
  )

Joins – Combining Datasets

Joins combine two data frames based on shared columns (keys).

Setup: Two Tables

# Roster of students
roster <- tibble(
  cadet_id = c(1, 2, 3, 4),
  name = c("Adams", "Baker", "Clark", "Davis")
)

# Grades (not everyone has a grade; some IDs don't match)
grades <- tibble(
  cadet_id = c(1, 2, 3, 5),
  grade = c(92, 85, 78, 96)
)

Join Types

# left_join: keep ALL rows from the left table
# Add grades where available; NA for Davis (id 4)
roster |> left_join(grades, by = "cadet_id")

# right_join: keep ALL rows from the right table
# Includes id 5 (not in roster); drops Davis
roster |> right_join(grades, by = "cadet_id")

# inner_join: keep ONLY rows that match in BOTH
# Only ids 1, 2, 3
roster |> inner_join(grades, by = "cadet_id")

# full_join: keep ALL rows from BOTH tables
# Includes everyone; NAs where no match
roster |> full_join(grades, by = "cadet_id")

Filtering Joins

These do not add columns – they filter rows.

# semi_join: keep rows in roster that HAVE a match in grades
roster |> semi_join(grades, by = "cadet_id")

# anti_join: keep rows in roster that DO NOT have a match in grades
roster |> anti_join(grades, by = "cadet_id")
# Returns Davis -- the student with no grade
Tipanti_join for data quality

anti_join() is your best friend for finding records that should match but don’t. Use it to find students missing grades, orders without customers, or sensor readings without a matching station ID.

When Key Names Differ

# If the key columns have different names
enrollments <- tibble(
  student_id = c(1, 2, 3),
  course = c("MA491", "MA491", "MA206")
)

roster |>
  left_join(enrollments, by = c("cadet_id" = "student_id"))

Join Decision Guide

You want… Use
All rows from the main table, add info if available left_join()
Only rows that exist in both tables inner_join()
All rows from both tables full_join()
Find which rows in A have no match in B anti_join()
WarningDuplicate key danger

If a key appears multiple times in the right table, your join will produce more rows than you started with. Always check nrow() before and after a join.

separate() and unite()

Split One Column into Multiple

messy <- tibble(
  date_location = c("2025-03-15_WestPoint", "2025-03-16_FortLiberty")
)

messy |>
  separate(date_location, into = c("date", "location"), sep = "_")

Combine Multiple Columns into One

clean <- tibble(
  year = c(2025, 2025),
  month = c(3, 4),
  day = c(15, 1)
)

clean |>
  unite("date", year, month, day, sep = "-")

separate_wider_delim() (newer tidyr)

# More explicit version
messy |>
  separate_wider_delim(date_location, delim = "_", names = c("date", "location"))

Real Messy Data Example

Here is a common pattern: data from a spreadsheet where the structure encodes information.

# Simulated PT test results -- wide format from a spreadsheet
pt_scores <- tibble(
  cadet = c("Adams", "Baker", "Clark"),
  pushups_fall = c(72, 58, 80),
  pushups_spring = c(78, 65, 82),
  run_fall = c(13.5, 15.2, 12.8),
  run_spring = c(13.1, 14.8, 12.5)
)

# Step 1: Pivot longer
pt_tidy <- pt_scores |>
  pivot_longer(
    cols = -cadet,
    names_to = c("event", "semester"),
    names_sep = "_",
    values_to = "score"
  )

pt_tidy

# Step 2: Now you can easily filter, group, and plot
pt_tidy |>
  ggplot(aes(x = semester, y = score, color = cadet, group = cadet)) +
  geom_point(size = 3) +
  geom_line() +
  facet_wrap(~event, scales = "free_y") +
  theme_minimal() +
  labs(title = "PT Score Changes by Semester")

The Tidy Data Checklist

Before you start any analysis, ask:

  1. Is each variable in its own column? (If not, pivot_longer())
  2. Is each observation in its own row? (If not, pivot_wider() or separate())
  3. Is each value in its own cell? (If not, separate())
  4. Do I need to combine tables? (If yes, pick the right join)
  5. After joining, did the number of rows change unexpectedly? (Check for duplicate keys)
